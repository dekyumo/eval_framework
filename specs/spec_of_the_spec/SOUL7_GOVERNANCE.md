It's 2026, AI governance is fashionable this year.

Let's take some elements of the NIST framework that could apply:
- govern 1: legal/compliance requirement and risk analysis is incorporated in the evals
- govern 2: users are indentified, evals make the test process obvious
- govern 3: bias monitoring
- govern 4: this
- govern 5: evaluation as a communication tool
- govern 6: robustness testing
- map 1: evals are set in the enterprise's context (mission, clients, actors)
- map 2: this
- map 3: scope and costs are evaluated
- map 4: risk and benefits of the agent are expressed as evals
- map 5: incidents create new evals
- measure 1: this
- measure 2: robustness/resilience/explainability/privacy/risk/fairness/sustainability are all addressed by the evaluation
- measure 3: evaluations must anticipate unknown risks
- measure 4: telemetry and observability
- manage 1: the TEVV process checks the agent, risks documented within the evals need managerial response planning
- manage 2: evals need to be updatable when a new risk factor comes up
- manage 3: underlying LLMs are evaluated
- manage 4: continuous improvement



=======

This influences the usage of an eval platform more than the eval platform itself, although all the above uses must be possible.
